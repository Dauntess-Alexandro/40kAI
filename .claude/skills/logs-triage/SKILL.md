# logs-triage — отладка 40kAI начинается с логов

Жёсткое правило проекта (AGENTS.md): **сначала логи → причина → потом фикс.**
Никаких «должно быть вот тут» без подтверждения из логов.

## Шаг 1. Выбери нужный лог
Все логи в `runtime/logs/`:

| Что сломалось | Файл |
|---------------|------|
| Обучение (`train.py`, self-play, чекпойнты) | `LOGS_FOR_AGENTS_TRAIN.md` |
| Игра / Viewer | `LOGS_FOR_AGENTS_PLAY.md` |
| Оценка (`eval.py`) | `LOGS_FOR_AGENTS_EVAL.md` |
| Remote inference server на ПК2 | `LOGS_FOR_AGENTS_REMOTE_IS.md` (+ `gmz_remote_is_*.log`) |

Не уверен в источнике — посмотри несколько и сопоставь по времени.

## Шаг 2. Прочитай СВЕЖИЕ строки
Читай конец файла (последние ~150–300 строк), а не весь файл. Цель — найти
последние события перед сбоем, а не историю с начала.

## Шаг 3. Найди маркеры
Грепай по фазам/ошибкам и распределённым тегам:
- ошибки: `Traceback`, `Error`, `ERROR`, `WARN`, `Exception`, `assert`, `invalid`, `stale_drop`.
- фазы/ходы движка: фаза хода, ход, юнит, цель, `advance`, `charge`, `no targets`.
- распределёнка/IS: `[AZ][INF_SERVER]`, `[AZ][ENV_WORKER]`, `[AZ][REMOTE_IS]`,
  `[AZ][REMOTE_CLIENT]`, `[AZ][DIST]`, `[AZ][DIST][RECEIVER]`, `[AZ][DIST][SINK]`.

## Шаг 4. Свяжи лог с кодом
По маркеру найди место в коде (`core/engine/`, `core/models/`, `core/envs/`).
Проверь телеметрию (CPU/RAM/GPU) рядом, если сбой похож на ресурсный.

## Шаг 5. Сформулируй вывод (русский, формат проекта)
- **Что случилось:** симптом + цитата из лога (`файл лога`, ключевая строка).
- **Где:** `файл.py:строка` (функция) — предполагаемая причина.
- **Что делать дальше:** конкретные шаги фикса/проверки.

Если в логах «тихий» пропуск без причины — это само по себе дефект логирования
(AGENTS.md): отметь, что лог должен объяснять, ПОЧЕМУ что-то пропущено.

## Дальше
Нашёл причину → передавай в `superpowers:systematic-debugging` (строгий цикл
гипотеза→проверка) и `superpowers:test-driven-development` (тест на регрессию до фикса).
**Сам лог-файл не правь руками** — его пишет рантайм (хук это и заблокирует).
