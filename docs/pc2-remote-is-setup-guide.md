# Гайд: поставить Remote Inference Server на второй ПК (ПК2)

Пошаговая инструкция для схемы **два компьютера**: обучение на **ПК1**, GPU-инференс (MCTS) на **ПК2**.

Полная документация: [`remote-inference-server-gmz.md`](remote-inference-server-gmz.md).

---

## Что получится в итоге

| Машина | Что работает |
|--------|----------------|
| **ПК1** | Qt GUI, `train` (learner на GPU), **6 CPU env workers** |
| **ПК2** | Только `tools\pc2_remote_is.bat` → inference server на GPU |
| **Связь** | ZMQ порт **5555**, веса через **общую папку SMB** |

На ПК2 **не нужны** Qt, train, play, eval.

---

## Перед началом (чеклист)

- [ ] Оба ПК в **одной сети** (лучше кабель Gigabit, не Wi‑Fi).
- [ ] На **ПК2**: Windows, **NVIDIA** (например RTX 2060 Super), `nvidia-smi` работает.
- [ ] На **ПК2**: **Python 3.12+** с галочкой **Add to PATH**.
- [ ] На **ПК2** лежит **тот же репозиторий 40kAI**, что на ПК1 (`git pull` на обеих — одна версия).
- [ ] На **ПК1** уже хотя бы раз запускали GMZ / есть checkpoint или `latest_gmz_policy.pth`.

Запишите IP ПК2: на ПК2 в cmd → `ipconfig` → **IPv4** (например `192.168.1.50`).

## Часть A — ПК2 (сервер инференса)

### Шаг 1. Скопировать репозиторий

```bat
cd C:\
git clone https://github.com/Dauntess-Alexandro/40kAI.git
cd 40kAI
git pull
```

Или скопируйте папку `C:\40kAI` с ПК1 на ПК2 (USB/сеть).

### Шаг 2. Подключить `Z:` (SMB) — см. шаг 4 ниже

Сначала общая папка и диск **`Z:`**, потом bat на ПК2.

### Шаг 3. Запуск bat на ПК2 (конфиг создаётся сам)

```bat
cd C:\40kAI
tools\pc2_remote_is.bat
```

При **первом** запуске скрипт копирует шаблон в `runtime\state\pc2_remote_is_config.bat` с готовыми значениями:

- `GMZ_REMOTE_WEIGHTS_PATH=Z:\latest_gmz_policy.pth`
- `GMZ_REMOTE_SEARCH_CONFIG=Z:\gmz_remote_search_cfg.json`
- `GMZ_REMOTE_PORT=5555`, `GMZ_REMOTE_DEVICE=cuda:0`, firewall и batch — из шаблона

**Блокнот не открывается** — сразу идёт setup и запуск сервера.

Редактировать вручную только если диск не `Z:` или другой порт:

```bat
tools\pc2_remote_is.bat config
```

| Переменная | Когда менять |
|------------|----------------|
| `GMZ_REMOTE_INIT_WEIGHTS` | Только если на `Z:\` ещё нет `latest_gmz_policy.pth` — путь к `.pth` на ПК2 |
| `GMZ_REMOTE_AUTH_TOKEN` | Если включили токен в GUI на ПК1 |
| `GMZ_REMOTE_WEIGHTS_PATH` / `SEARCH_CONFIG` | Если буква диска не `Z:` |

Для **RTX 2060 Super (8 GB)** в шаблоне (как Heavy / Very Heavy на ПК1): `GMZ_REMOTE_BATCH_SIZE=20`, `GMZ_REMOTE_BATCH_INTERVAL_MS=5`, `GMZ_REMOTE_COMPILE=0`.

### Шаг 4. Общая папка (SMB) с весами

**На ПК1** (компьютер, где идёт train и пишутся веса):

1. Папка должна существовать (файл `latest_gmz_policy.pth` появится после первого sync при train; до train можно расшарить пустую папку).  
   Кнопка **«ОЧИСТИТЬ КЭШ»** удаляет `latest_gmz_policy.pth` и прочие файлы в `actor_sync`, но **саму папку** не трогает (чтобы не сбивать SMB).
   ```
   C:\40kAI\artifacts\models\actor_sync\
   ```

2. **Расшарить папку (Windows 10/11)** — делать **на ПК1**, в проводнике, по папке `actor_sync`.

   #### Способ 1 — проще всего (домашняя сеть, только чтение для всех)

   Где: проводник → `C:\40kAI\artifacts\models\` → папка **`actor_sync`**.

   1. **ПКМ** по `actor_sync` → **Предоставить доступ к** → **Определённые люди…**
   2. Выпадающий список справа вверху: выберите **Все** (англ. **Everyone**).
   3. Справа внизу уровень прав: **Чтение** (не «Чтение и запись»).
   4. Кнопка **Общий доступ**.
   5. Windows покажет сетевой путь, например `\\DESKTOP-XXX\actor_sync` — **скопируйте** (он понадобится на ПК2).

   Если пункта «Предоставить доступ к» нет: **ПКМ** → **Свойства** → вкладка **Доступ** → кнопка **Общий доступ…** → в списке **Все** → **Добавить** → уровень **Чтение** → **Общий доступ**.

   #### Способ 2 — только конкретный пользователь с ПК2

   **Где узнать логин на ПК2** (на втором компьютере):

   - `Параметры` → **Учётные записи** → **Ваши данные** → имя пользователя (локальная учётка),  
     или
   - Win+R → `cmd` → команда:
     ```bat
     whoami
     ```
     Пример вывода: `DESKTOP-PC2\Ivan` → для шаринга на ПК1 обычно нужно **`DESKTOP-PC2\Ivan`** (имя_ПК2\логин).

   **Где добавить на ПК1:**

   1. ПКМ по `actor_sync` → **Предоставить доступ к** → **Определённые люди…**
   2. Кнопка **Добавить…** (или поле ввода внизу).
   3. Введите `DESKTOP-PC2\Ivan` (подставьте своё из `whoami` на ПК2) → **Проверить имена** → **ОК**.
   4. Выделите добавленного пользователя → справа **Чтение** → **Общий доступ**.

   Если «имя не найдено»: на ПК1 в том же окне можно временно использовать **Все** (способ 1). Либо при подключении диска с ПК2 вводите логин/пароль **учётки ПК1** (см. шаг 5 ниже).

   #### Расширенный общий доступ (если нужен именно «классический» SMB-share)

   1. ПКМ → **Свойства** → **Доступ** → **Расширенная настройка…**
   2. Включить **Открыть общий доступ к этой папке**.
   3. **Разрешения** → **Добавить…** → ввести `Все` или `DESKTOP-PC2\Ivan` → **Проверить имена** → **ОК**.
   4. Для этой записи отметить только **Чтение** (галочка «Изменение» не нужна) → **ОК** → **ОК**.
   5. На вкладке **Доступ** кнопка **Общий доступ…** — как в способе 1, чтобы увидеть путь `\\…\actor_sync`.

3. Узнайте имя ПК1: `Параметры` → **Система** → **О сведении** → **Имя устройства** (например `DESKTOP-ABC`).  
   Или в cmd на ПК1: `hostname`

4. Сеть: оба ПК в одной Wi‑Fi/LAN; на ПК1 включите **Сетевое обнаружение**  
   (`Параметры` → **Сеть и Интернет** → **Доп. параметры сети** → **Включить** обнаружение и общий доступ).

**На ПК2** (inference server) — подключить общую папку ПК1 как диск **`Z:`**.

#### Перед подключением (на ПК2)

1. Оба ПК в **одной Wi‑Fi / LAN** (не гостевая сеть роутера).
2. На **ПК1** train/SMB уже настроен; путь с ПК1 вы видели, например: `\\USER-PC\actor_sync`.
3. Узнайте **IP ПК1** (если по имени не откроется): на **ПК1** в cmd:
   ```bat
   ipconfig
   ```
   Строка **IPv4-адрес** (например `192.168.0.15`). Тогда путь: `\\192.168.0.15\actor_sync`.

4. Проверка с **ПК2** в cmd (замените имя или IP):
   ```bat
   ping USER-PC
   ```
   или `ping 192.168.0.15` — должны идти ответы.  
   В проводнике ПК2 в адресной строке введите `\\USER-PC` — должен открыться список шаров; клик по **`actor_sync`** — видно содержимое папки (может быть пусто).

#### Способ A — через проводник (пошагово)

1. Откройте **Проводник** (Win+E).
2. Слева **Этот компьютер** (или «Компьютер»).
3. Вверху вкладка **Компьютер** (иногда **Средства работы с дисками**) → кнопка **Подключить сетевой диск**.  
   *Если кнопки нет:* ПКМ по **Этот компьютер** → **Подключить сетевой диск…*
4. Откроется окно **«Подключение сетевого диска»**:
   - **Диск:** выпадающий список → выберите **`Z:`** (любая **свободная** буква; если `Z:` занят — `Y:` и т.д., тогда в конфиге путь будет `Y:\…`).
   - **Папка:** введите **точно** (слэши как в примере, без `C:`):
     ```text
     \\USER-PC\actor_sync
     ```
     Подставьте **имя ПК1** из шага расшаривания (у вас было `USER-PC`) или IP:
     ```text
     \\192.168.0.15\actor_sync
     ```
   - **Восстанавливать подключение при входе в систему** — по желанию (удобно, чтобы после перезагрузки ПК2 диск `Z:` поднимался сам; если Windows забудет пароль — снимите галочку и подключайте вручную).
   - **Использовать другие учётные данные** — включите, если без галочки не пускает или просит пароль каждый раз.
5. **Готово** / **Завершить**.
6. Если появится **«Введите сетевые учётные данные»**:
   - Это логин **ПК1**, не ПК2.
   - Имя пользователя, варианты:
     - `USER-PC\Professional` — `ИМЯ-ПК1\логин` (логин с ПК1, у вас в шаринге был **Professional**);
     - или только `Professional`, если Windows сам подставит компьютер.
   - Пароль — **пароль входа в Windows на ПК1** для этого пользователя.
   - Можно отметить **Запомнить учётные данные**.
7. В проводнике слева появится **Сетевой диск (Z:)** → откройте — это содержимое `actor_sync` с ПК1.

#### Способ B — через cmd (если GUI не сработал)

На **ПК2** от имени обычного пользователя (при необходимости — «Запуск от имени администратора»):

```bat
net use Z: \\USER-PC\actor_sync /user:USER-PC\Professional ВАШ_ПАРОЛЬ_ПК1 /persistent:yes
```

- `Z:` — та же буква, что в `GMZ_REMOTE_WEIGHTS_PATH`.
- `/persistent:yes` — как «восстанавливать при входе».
- Пароль в cmd виден в истории — для постоянной работы лучше способ A с «Запомнить».

Отключить диск: `net use Z: /delete`

#### Проверка на ПК2

В cmd:
   ```bat
   dir Z:\
   dir Z:\latest_gmz_policy.pth
   ```
   Пока train не запускали, второй файл может отсутствовать — тогда на ПК2 в конфиге укажите **`GMZ_REMOTE_INIT_WEIGHTS`** (checkpoint с ПК1) для первого старта.

#### Если не подключается

| Симптом | Что сделать |
|--------|-------------|
| «Сетевой путь не найден» | `ping` ПК1; открыть `\\USER-PC` в проводнике; попробовать IP вместо имени; на ПК1 не спать, Wi‑Fi одна сеть. |
| «Отказано в доступе» | На ПК1 в шаре есть **Все** с **Чтением**; ввести `USER-PC\Professional` + пароль ПК1; на ПК1: **Параметры сети** → профиль **Частная**. |
| Просит гость / не принимает пароль | На ПК1: **Панель управления** → **Центр управления сетями** → **Изменить доп. параметры общего доступа** → включить обнаружение и общий доступ; **отключить** «общий доступ с парольной защитой» *или* завести на ПК1 локального пользователя с тем же логином/паролем, что на ПК2 (редко нужно). |
| Диск `Z:` пропал после перезагрузки | Снова **Подключить сетевой диск** или `net use … /persistent:yes`; ПК1 должен быть включён. |

В `pc2_remote_is_config.bat` на ПК2:
```bat
set "GMZ_REMOTE_WEIGHTS_PATH=Z:\latest_gmz_policy.pth"
```

### Шаг 5. Файл `gmz_remote_search_cfg.json`

**На ПК1** (после настройки ростера в GUI и `hyperparams.json`):

```bat
cd C:\40kAI
tools\write_gmz_remote_search_cfg.bat
```

Скрипт читает `runtime/state/data.json` (ростер из GUI) и `hyperparams.json` → `gumbel_muzero`, строит env как при train и пишет **два файла**:

| Путь на ПК1 | Назначение |
|-------------|------------|
| `runtime\state\gmz_remote_search_cfg.json` | локально / опционально git |
| `artifacts\models\actor_sync\gmz_remote_search_cfg.json` | **та же SMB-папка**, что и веса — видна на ПК2 как `Z:\` |

**На ПК2** в `pc2_remote_is_config.bat` (после подключения диска `Z:`):

```bat
set "GMZ_REMOTE_SEARCH_CONFIG=Z:\gmz_remote_search_cfg.json"
```

Проверка с ПК2: `dir Z:\gmz_remote_search_cfg.json`

Git push/pull для JSON **не обязателен**, если ПК2 читает конфиг с `Z:\`. Удобно держать копию в репо — по желанию.

**Важно:** после смены ростера, миссии или GMZ на ПК1 — **запустите bat снова** (обновятся оба файла + `Z:\` на ПК2), перезапустите remote IS на ПК2.

Ручной JSON не нужен; поля `_generated_utc` / `_sources` в файле — служебные, сервер их игнорирует.

Подробнее: [`remote-inference-server-gmz.md` §6](remote-inference-server-gmz.md#6-файл-search_cfgjson).

### Шаг 6. Установка зависимостей и запуск сервера

Снова:

```bat
cd C:\40kAI
tools\pc2_remote_is.bat
```

Первый полный прогон:

- вызовет `installer\install_deps.bat -y` (**5–20 мин**);
- поставит PyTorch+CUDA, **pyzmq**, **msgpack**;
- откроет firewall на порт 5555;
- запустит сервер.

**Успех** — в консоли:

```text
[GMZ][REMOTE_IS] listening on 0.0.0.0:5555 device=cuda
```

Окно **не закрывайте** — пока оно открыто, ПК2 принимает запросы. Остановка: **Ctrl+C**.

Проверка только окружения (без сервера):

```bat
tools\pc2_remote_is.bat check
```

Логи: `runtime\logs\gmz_remote_is_<дата>.log`.

## Часть B — ПК1 (обучение)

### Шаг 7. Обновить код и venv

```bat
cd C:\40kAI
git pull
.\.venv\Scripts\activate
python -m pip install -U pyzmq msgpack
```

(Или полный `installer\install_deps.bat -y`.)

### Шаг 8. Настройки GUI

1. Запустите Qt GUI.
2. **Настройки** → **Gumbel MuZero** → панель **Inference Server**.
3. Выключите **Local Inference server**.
4. Включите **LAN Inference server**.
5. **Хост ПК2** = IP из `ipconfig` на ПК2 (например `192.168.1.50`), **не** `127.0.0.1` (если ПК2 — другая машина).
6. **Порт** = `5555`.
7. **Веса на ПК2** = тот же SMB-путь, что видите с ПК1 (для справки).
8. **Проверить соединение** → должно быть **OK • GPU: ...**.
9. **Сохраните** hyperparams.

### Шаг 9. Запуск train

1. На ПК2 сервер **уже запущен** (`pc2_remote_is.bat`).
2. **Главная** → алгоритм **Gumbel MuZero** → **Тренировка**.

Если LAN включили случайно на одном ПК — в логе train будет подробная подсказка, что выключить LAN и включить Local.

## Частые проблемы

| Симптом | Решение |
|---------|---------|
| Кракозябры в `install_deps.bat` (`╨Ъ╨╛╤А...`) | Нормально для старого **cmd.exe**: запустите `chcp 65001`, используйте **Windows Terminal**, или `installer\install_deps.bat -y` (меню на английском). На работу Python это не влияет. |
| `Connection refused` | На ПК2 запущен bat? Firewall? Верный IP? |
| `search_cfg не найден` | Создайте JSON, путь в конфиге |
| `Нет файла весов` | SMB: `dir Z:\latest_gmz_policy.pth` с ПК2 |
| Пустой пароль / запрещён вход | На **ПК1** задать пароль учётки; на **ПК2** `net use Z: \\USER-PC\actor_sync /user:USER-PC\Professional пароль_ПК1` |
| `triton` not installed | Не ошибка: на Windows поставьте `GMZ_REMOTE_COMPILE=0` в конфиге или обновите код (compile пропускается сам) |
| HealthCheck OK, train падает | Одна версия кода на ПК1 и ПК2 (`git pull`) |
| Один ПК, случайно LAN | Выключите LAN, включите Local |

## RTX 2060 Super на ПК2

- **VRAM 8 GB** — для remote IS (только inference, без learner) **достаточно**.
- Узкое место чаще **скорость GPU + LAN**, не память.
- Начните с preset **Balanced** на ПК1; при стабильности — Heavy.
