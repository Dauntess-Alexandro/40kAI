# Редизайн вкладки «Метрики модели» — вариант A «Одним взглядом»

Дата: 2026-06-13
Статус: согласован, готов к плану реализации

## Цель

Видеть состояние обучаемой модели сразу, без скролла и без повторяющихся блоков.
Структура остаётся одной вкладкой (`Метрики модели` в `Main.qml`), меняется только порядок и
устраняются дубли. Новой функциональности и зависимостей не вводим.

## Проблемы текущей раскладки (что чиним)

1. **Двойное дублирование данных:**
   - верхняя карточка «Последнее окно тренировки» (ep / winrate / reward / ep_len) ≈ KPI-плитки ниже;
   - верхняя карточка «Модель» (algo / mode / run id) ≈ повторяет данные, доступные в шапке.
2. **Шум в шапке:** английский заголовок `Model Metrics` на русской вкладке + длинный абзац-описание.
3. **Нет иерархии:** 8 графиков сеткой 2×4 одинаковой плотности — «всё важно = ничего не важно».
4. **Длинный скролл:** чтобы дойти до графиков, нужно пройти три блока шапки.

## Итоговая раскладка (сверху вниз)

### 1. Шапка — одна строка
- Слева: заголовок `Метрики модели` + чипы `model_<id> · <ALGO>`, `selfplay/off`.
- Справа: кнопки `Выбрать` / `Последняя` / `TensorBoard ↗`.
- Английский `Model Metrics` удаляется.
- Длинный абзац-описание («Метрики тренировки: каждая точка — окно…») не теряется: переносится в
  тултип у иконки «?» рядом с заголовком.

### 2. Блок «Состояние» — 4 KPI-плитки
`WINRATE` · `REWARD` · `ДЛИНА ЭПИЗОДА` · `ОКНО (ЯКОРЬ)`.
Единственное место с этими числами. Значения — из существующих
`controller.detWinrateLast / detRewardLast / detEpLenLast / detEpisodeLast`.

### 3. «Главные кривые» — 2 крупных графика
Winrate и Reward в две колонки (по факту крупнее текущих за счёт ширины).
Компонент `MetricChart.qml` переиспользуется как есть, включая чипы тек/ср/мин/макс.

### 4. ▸ «Остальные метрики» — сворачиваемый блок, **по умолчанию свёрнут**
Сетка 2×3 из оставшихся спеков: Loss, Ep_len, Avg VP, HP diff, Kill diff, Причины завершения.

### 5. ▸ «Детали модели и оппонента» — сворачиваемый блок, **по умолчанию свёрнут**
Справочная инфа из старых карточек:
- Модель: algo / mode / run id;
- Оппонент: self-play / источник оппонента / алгоритм оппонента (+ id).

## Что удаляется (дубли)

- Верхняя карточка «Последнее окно тренировки» — заменена KPI-плитками.
- Верхняя карточка «Модель» в summary-строке — данные ушли в чипы шапки и блок «Детали».

## Что НЕ трогаем (YAGNI)

- Источник данных: `controller.detSeries()`, `chartSpecs`, сигнал `onMetricsSummaryChanged`.
- Компонент `MetricChart.qml` (логика и рендер Canvas без изменений).
- Пустое состояние «Нет данных метрик» — сохраняется (показывается вместо графиков при `count === 0`).
- Вкладки «Метрики эвристики» (`HeurMetricsPanel`) и «Играть».

## Технические заметки

- Все правки — в `Main.qml`, блок вкладки «Метрики модели» (ориентировочно строки ~3874–4191).
- Сворачиваемые секции реализуются на существующем `ChamferPanel` + кликабельный заголовок-строка
  (свой мини-expander через `property bool expanded` и `visible`-биндинг контента). Новых
  QML-импортов/зависимостей не требуется.
- Стиль карточек/токены берём из уже используемых `root.*` (bgElevated, borderMuted, accent*, uiText*).

## Критерии готовности

- На свежем запуске вкладки видны только: шапка (1 строка) + 4 KPI + 2 главных графика;
  оба сворачиваемых блока закрыты.
- Раскрытие «Остальные метрики» показывает 6 графиков; раскрытие «Детали» — справочные поля.
- Нет повторяющихся числовых блоков (winrate/reward/len/episode — ровно один раз).
- При отсутствии данных показывается прежнее пустое состояние, без падений.
- Поведение «Метрики эвристики» и «Играть» не изменилось.
