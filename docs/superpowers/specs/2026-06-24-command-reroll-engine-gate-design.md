# Движковый гейт command_reroll (нет цели → нелегально) — дизайн

Дата: 2026-06-24
Статус: согласован заказчиком

## Цель

Не предлагать стратагему **command_reroll**, если у юнита нет предстоящего броска в этой фазе
(нет валидной цели стрельбы / чарджа). Тогда реролл становится **нелегальным в action-масках**
для всех алгоритмов (DQN/PPO/AZ/GMZ/SMZ), и исчезает доминирующий «no-target» waste (трата CP
впустую). Структурный фикс на уровне движка — работает и для MuZero, который игнорирует
шейпинг-reward (учится только на win/loss).

## Контекст (факты из кода)

- command_reroll применяется проактивно: добавляет эффект в `active_stratagem_effects`; «fires»
  (`_cmd_reroll_fired += 1`) только когда есть реальный бросок hit/wound в нужной фазе, который
  его потребляет (`core/envs/warhamEnv.py:2407-2415`, `:2427-2450`). «Wasted» = применили, но
  броска не случилось.
- Реролл предлагается в `command_reroll_options_for_unit` (`core/engine/phases/option_generator.py:129`),
  гейт только по: фаза содержит стратагему, хватает CP, юнит жив. **Не проверяет, будет ли бросок.**
- Окна реролла строятся только если опции непусты (`option_generator.py:246-250` shooting,
  `:272-277` charge). Пустой список → окно не создаётся → действие нелегально в масках.
- Готовые предикаты цели: `e.get_shoot_targets_for_unit(side, unit_idx)` (исп. в
  `shooting_options_for_unit:31`), `e.get_charge_targets_for_unit(side, unit_idx)`
  (`charge_options_for_unit:99`). Пустой список = у юнита нет цели/броска.

## Решения заказчика

- Предикат: **нет валидной цели → реролл нелегален** (по `get_*_targets_for_unit`).
- Scope: **Shooting (hit/wound) + Charge, обе стороны** (функция берёт `side`). FIGHT-реролл
  сейчас не предлагается — не трогаем.

## Не-цели (YAGNI)

- Случай «цель есть, но модель решит не стрелять» — не гейтим (нужно предсказывать намерение).
- Изменение reward/штрафов — нет (это для DQN/PPO, отдельно).
- FIGHT-фаза реролла — нет (не предлагается).

## Архитектура

Один локальный гейт в начале `command_reroll_options_for_unit`:

```
eligible:
  Phase.SHOOTING -> bool(e.get_shoot_targets_for_unit(side, unit_idx))
  Phase.CHARGE   -> bool(e.get_charge_targets_for_unit(side, unit_idx))
  иначе          -> True   # защитно, прочие фазы не трогаем
if not eligible: return []
```

Ставится ПОСЛЕ `_unwrap` и существующих проверок фазы/CP/живости (или до них — порядок не важен,
т.к. при no-target всё равно `[]`). Поведение: нет цели → опций нет → DecisionWindow реролла не
строится → в legal-маске реролла нет → ни один алго его не выберет.

## Обработка краёв

- Юнит мёртв / мало CP / фаза без стратагемы → как сейчас (`[]`), гейт не мешает.
- `get_*_targets_for_unit` кидает/недоступен (не должно) → дизайн полагается на существующее
  поведение этих методов (они уже вызываются в shooting/charge options); доп. обработки не вводим.
- Неизвестная фаза → `eligible=True` (не сужаем легальность вне scope).

## Риски

- Изменение legal-масок может затронуть существующие тесты, ожидающие реролл при no-target →
  сверка baseline это ловит; такие тесты, если есть, отражают старое (баг) поведение — обновить
  при необходимости с пометкой.
- Эффект на обучение: меньше «мусорных» действий в пространстве → чище сигнал; винрейт не должен
  упасть (реролл без цели и так был бесполезен).
