# Stage 8 — Roadmap: подключение PhaseEngine/стратагем к обучению (RL/Zero)

**Дата:** 2026-06-17
**Ветка:** `feat/phase-decision-windows`
**Статус:** роудмап (декомпозиция, без кода). Каждый под-проект → отдельная спека → план → TDD.

## Контекст: где мы сейчас

Построен полный аддитивный слой фаз/стратагем, **поведение игры 1:1** (golden-trace):
- `core/engine/phases/`: типы окон, генератор опций, legacy-компилятор, реестр (7 стратагем), `apply` + журнал, `PhaseEngine.run_command/movement/shooting/charge/fight`.
- `attack()` opt-in эффекты (cover/reroll_hits/reroll_wounds/reroll_save/strength_mod/ap_improve).
- Рабочие стратагемы end-to-end: Insane Bravery, Hungry Void, Command Re-roll (approx), Go to Ground; реакции через `reaction_policy`.

**Ключевой разрыв:** обучение (`train.py`, MCTS, replay, 6 алгоритмов) всё ещё гоняет **плоский `env.step`**. PhaseEngine — параллельный путь, агент им не пользуется. Агент **не видит** CP/стратагемы в observation и **не выбирает** их.

## Цель Stage 8

Замкнуть петлю: агент **видит** состояние стратагем/CP и **выбирает** стратагемы как часть политики, обучение использует это. Конечная точка — RL/Zero-агент осмысленно тратит CP.

## Текущая интеграция обучения (факты из кода)

- **Observation:** `get_observation_for_side(side)` — вектор `[health,x,y]×model, CP_model, [health,x,y]×enemy, CP_enemy, objectives×2, game_over]`. Размер фиксирован в `observation_space` (`warhamEnv.py:1121`). CP уже есть; фазы/доступных стратагем/журнала — нет.
- **Action:** факторизованный плоский контракт `core/models/action_contract.py` (`BASE_ACTION_HEADS + move_num_i`). `use_cp`(0..4)/`cp_on` — головы для command-bravery; активные стратагемы (Hungry Void и т.п.) в плоском контракте НЕ выражаются.
- **AZ/GAZ:** `play_episode_with_mcts` (`alphazero_selfplay.py`) → `get_legal_action_masks_by_head` → `mcts.run` (joint-кандидаты через `snapshot_state`/`env.step`/`enemyTurn`/`restore`) → `convertToDict` → `env.step`. Узел дерева = целый раунд.
- **Replay:** `AZTransition(state, policy_targets, value_target, policy_version, faction)` — без phase/stratagem-метаданных.
- **PPO/DQN:** `ActorCriticMultiHead` — по голове на ключ действия.
- **Алгоритмы:** DQN, PPO, AlphaZero(tree), Gumbel AZ, Gumbel MuZero, Sampled MuZero — общая факторизованная парадигма; AZ-семейство делит инфру (`az_*`/`gmz_*`).

## Декомпозиция на под-проекты (порядок по риску/ценности)

### 8.1 — Observation-фичи фаз/CP (низкий риск, фундамент)
**Что:** добавить в observation: one-hot текущей фазы, флаг timing(main/reaction), CP обеих сторон (есть, но нормализовать), флаги «доступна стратагема X сейчас», счётчики из журнала `stratagem_used` (использовано в этом ходу/фазе).
**Зачем:** агент не может научиться использовать CP, если не «видит» его контекст.
**Риск:** меняет размер obs → несовместимо со старыми чекпойнтами. Митигейт: фиче-флаг (env-переменная) `PHASE_OBS_FEATURES=0/1`, дефолт 0 (старый размер); версионирование контракта (`build_env_contract_from_spaces` уже есть). Поведение игры не меняется.
**Тесты:** размер obs при флаге 0 == старому; при флаге 1 содержит ожидаемые поля; детерминизм.
**НЕ трогает:** action space, MCTS, обучение по умолчанию.

### 8.2 — Replay-метаданные (низкий риск, обучения не меняет)
**Что:** расширить `AZTransition` (и аналоги в gmz/smz replay) опциональными полями: `phase`, `sub_step`, `window_id`, `chosen_option`, `stratagem_id|None`, `cp_before/after`. Заполнять при self-play, если доступно; дефолт None.
**Зачем:** отладка + будущее обучение «по окнам»; данные собираются заранее.
**Риск:** низкий (опциональные поля, дефолт None). Митигейт: обратная совместимость сериализации.
**Тесты:** старые трансформации читаются; новые поля round-trip.

### 8.3 — Командные стратагемы через существующие головы (средний риск)
**Что:** PhaseEngine-режим self-play, где command-окно (Insane Bravery) реально приходит от политики через decision-seam (а не из плоского `use_cp`). Подключить `decide_bravery` к политике в AZ self-play **за фиче-флагом**, исполнение через `PhaseEngine.run_command`.
**Зачем:** первый раз агент *решает* стратагему в обучении (на самой простой — командной).
**Риск:** трогает self-play петлю. Митигейт: флаг `WINDOWED_SELFPLAY=0/1`, дефолт 0 (старый путь); golden-trace + winrate-baseline в eval ДО включения.
**Тесты:** windowed self-play на флаге 0 == старому; на флаге 1 командные решения идут от политики.

### 8.4 — Активные стратагемы как ActionOption в MCTS-кандидатах (высокий риск, ядро цели)
**Что:** корневые/узловые кандидаты MCTS строятся из `generate_windows`/`fight_stratagem_options_for_unit` (list[ActionOption]), а не из декартова произведения плоских голов. Проекция факторизованных приоров на опции. Исполнение — через `PhaseEngine.run_*`. Узлы дерева = decision-окна (или хотя бы фазовые окна).
**Зачем:** только так агент/поиск выбирает активные стратагемы (Hungry Void/Command Re-roll) и тонкие per-unit решения.
**Риск:** высокий — переписывает кандидат-генерацию и шаг симуляции MCTS. Митигейт: один алгоритм первым (AlphaZero tree, уже на snapshot/MCTS), фиче-флаг, golden-trace, A/B winrate. Делать пофазно (сначала command+fight-окна, потом movement/shoot/charge).
**Тесты:** кандидаты = легальные опции; симуляция не течёт CP/phase (расширение snapshot-инварианта); winrate ≥ baseline.

### 8.5 — Перенос остальных алгоритмов (средний риск, объём)
**Что:** распространить windowed-режим на Gumbel AZ → Gumbel MuZero → Sampled MuZero → PPO/DQN (последние — либо через компилятор per-turn, либо опциональный option-режим).
**Зачем:** единый контракт во всех ветках; избежать рассинхрона (`az_*`↔`gmz_*`).
**Риск:** объём + рассинхрон параллельных реализаций. Митигейт: `code-reviewer`-субагент (ищет рассинхрон), общий код в `core/engine/phases/`, по одному алгоритму за заход.
**Тесты:** per-algo smoke + winrate.

### 8.6 — Долги/корректность (низкий риск, попутно)
- `expected_damage()` выровнять под новые `attack`-эффекты (сейчас EV их игнорирует → эвристика врага недооценивает баффнутые атаки).
- CP по правилам 10ed (Heroic 2→1 и проверка прочих) — поведенческое, отдельно, под golden-trace re-baseline.
- Hungry Void: добавить AP-улучшение для CHARACTER (сейчас только +S).

## Safety-harness (обязателен перед 8.3+)

1. **golden-trace** (есть) — поведение env.step.
2. **winrate-baseline** (добавить): снять эталонный winrate на N эпизодов фикс. сидами ДО windowed-изменений; сверять после. Это ловит сдвиг обучения, который golden-trace не видит.
3. **snapshot/restore инвариант** — расширить на новые поля (`active_stratagem_effects` уже покрыт).

## Рекомендованный порядок

**8.1 → 8.2 → 8.3 → (winrate-baseline) → 8.4 (AlphaZero) → 8.5 → 8.6 попутно.**
Обоснование: сначала безопасный фундамент (агент видит CP) и сбор данных (replay), потом самая простая интеграция решений (command), и только с harness — рискованное ядро (MCTS-кандидаты), затем тиражирование.

## Риски (сводно)

- **Сдвиг обучения** не виден golden-trace → нужен winrate-baseline (harness п.2).
- **Несовместимость obs/чекпойнтов** → фиче-флаги + версионирование контракта.
- **Рассинхрон 6 алгоритмов** → общий код + code-reviewer, по одному за заход.
- **Переменная длина policy-таргета (опции)** → до 8.4 факторизованная сеть + проекция; смена формата осознанно.
- **Производительность MCTS** (окна = больше узлов/инференсов) → батч-leaf-eval (есть), кэш приоров, лимит reaction-окон.
