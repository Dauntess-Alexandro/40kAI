# Spec: Редизайн TrainingAlgoHelpDialog → стиль E1 (датащит)

**Дата:** 2026-06-17  
**Статус:** утверждён, готов к планированию

---

## 1. Контекст и цель

Существующий `TrainingAlgoHelpDialog.qml` открывается кнопкой «?» рядом с выбором алгоритма (`Main.qml:1870`). Навигация — горизонтальные табы (`TabBar` + `StackLayout`). Внутри каждого таба — `AlgoHelpCard` (бейджи + секции с подробным текстом) + `AlgoCompareTable`.

Цель: сменить навигацию на вертикальный сайдбар по семействам и добавить датащит-шапку с 4 стат-барами — как в утверждённом мокапе E1 (`mockups/model_help_E_variants.html`). Весь существующий текстовый контент и таблица сравнения сохраняются.

---

## 2. Что меняем

### 2.1 Раскладка диалога

**Было:**
```
[ сводные карточки (2 шт.) ]
[ TabBar: DQN | PPO | AZ Tree | AZ Proxy | GMZ | SMZ | GAZ | Сравнение ]
[ StackLayout → AlgoHelpCard / AlgoCompareTable ]
[ Закрыть ]
```

**Станет:**
```
[ сводные карточки (2 шт.) ]
[ Row:  сайдбар (фикс. ширина)  |  ScrollView → контент  ]
[ Закрыть ]
```

`RowLayout` делит пространство: сайдбар ~180 px × uiScale, контент — `Layout.fillWidth: true`.

### 2.2 Сайдбар (`AlgoHelpSidebar.qml` — новый компонент)

Вертикальный `ColumnLayout`. Внутри — группы через `Repeater` по семействам:

| Группа      | Пункты                                 |
|-------------|----------------------------------------|
| Off-policy  | DQN                                    |
| On-policy   | PPO                                    |
| AlphaZero   | AZ Tree · AZ Proxy · GAZ               |
| MuZero      | GMZ · SMZ                              |
| _(разделитель)_ | **Сравнение** (служебный пункт)    |

Каждый пункт — `Rectangle` + цветной маркер-ромб + `Text`. Активный пункт: фон `accentColor` с opacity 0.15, левая полоска 2 px в `accentColor`.

Сайдбар принимает `property int currentIndex` и сигнал `indexChanged(int idx)`. Отображение порядка индексов совпадает с текущим `StackLayout` (0=DQN … 6=GAZ, 7=Сравнение).

### 2.3 Датащит-шапка в `AlgoHelpCard.qml`

Добавляем к существующим свойствам:

```qml
property string role: ""      // "Rainbow-стрелок", "Тактик-предиктор", …
property string abbr: ""      // "DQN", "AZ", "GMZ", …
property var stats: []        // [{label, value}, …] — 4 оси
```

Шапка рендерится **перед** текущим контентом карточки:

```
[ сигил-ромб (abbr, цвет акцента) ]  [ algoTitle  ]
                                       [ Семейство · role ]
                                       [ tldr (italic) ]
[ --- стат-бары (4 оси) --- ]
[ чипы/бейджи ]
[ --- существующие секции --- ]
```

**Сигил:** внешний `Rectangle` 48×48 px с `rotation: 45; clip: true`, внутри контр-ротированный `-45°` `Text` с `abbr`. Простое QML-решение без Canvas.

**Стат-бары (4 оси):**

| Ось | DQN | PPO | AZ Tree | AZ Proxy | GAZ | GMZ | SMZ |
|-----|-----|-----|---------|----------|-----|-----|-----|
| Скорость старта  | 4 | 5 | 2 | 3 | 2 | 1 | 2 |
| Сила тактики     | 4 | 2 | 5 | 3 | 4 | 5 | 4 |
| Цена инференса   | 1 | 1 | 4 | 2 | 4 | 5 | 3 |
| Сложность        | 4 | 3 | 4 | 3 | 5 | 5 | 4 |

Значение 1–5. Бар: `Rectangle` (фон `#1a2742`) + вложенный `Rectangle` шириной `value/5 * trackWidth`, цвет `#3f7bff` → `#7fa6ff` (горизонтальный gradient через `ShaderEffect` или просто один цвет `#5a8fff`). Рядом подпись `value/5` моноширинным шрифтом.

Существующие секции (Как учится / Сильные стороны / Ограничения / Режимы инференса / Температура / Когда выбирать) остаются **без изменений**.

### 2.4 `TrainingAlgoHelpDialog.qml`

- Убрать `TabBar` и `AlgoHelpTabButton` (компонент оставляем, просто не используем).
- Добавить `AlgoHelpSidebar` + `RowLayout` для сайдбар + контент.
- Добавить свойства `role`, `abbr`, `stats` к каждому `AlgoHelpCard` в `StackLayout`.
- Сводные карточки сверху — оставить без изменений.

### 2.5 Что НЕ трогаем

- `AlgoCompareTable.qml` — без изменений, пункт «Сравнение» открывает его как раньше.
- `AlgoHelpTabButton.qml` — оставляем (не удаляем), просто не используем в диалоге.
- Весь текст секций в `TrainingAlgoHelpDialog.qml` — без изменений.
- Проводка в `Main.qml` (`initialAlgo`, `trainingAlgoHelpDialog.open()`) — без изменений.
- `main.py`, `hyperparams.json` — не трогаем.

## 6. Проверка готовности

1. GUI запускается без ошибок QML.
2. Кнопка «?» открывает диалог; активный алгоритм подсвечен в сайдбаре.
3. Клик по каждому пункту сайдбара показывает нужную карточку.
4. В каждой карточке видны сигил + 4 стат-бара + существующие секции.
5. Пункт «Сравнение» открывает таблицу.
6. Диалог закрывается кнопкой «Закрыть».
