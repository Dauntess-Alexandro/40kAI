# 40kAI — CLAUDE.md

> RL-песочница по Warhammer 40k: движок (`core/`), обучение `train.py` (DQN, PPO, AlphaZero Tree/Proxy, Gumbel MuZero),
> оценка `eval.py`, Viewer (`app/viewer/`), Qt GUI на PySide6+QML (`app/gui_qt/`). Платформа: **Windows**, Python **3.12+**.
> Распределённое обучение и remote inference server (ПК1+ПК2).

## Плагины маркетплейса — когда задействовать

Установлены и включены; используй там, где помогают:

- **`pyright-lsp`** — навигация по типам и ловля ошибок до запуска. Опирайся на него при работе с большим модульным кодом (`core/models/az_*`, `gmz_*`, движок).
- **`serena`** — семантический анализ/безопасный рефакторинг через MCP. Для навигации и рефакторинга дублирующихся протоколов (`az_inference_*` / `gmz_inference_*`).
- **`context7`** — актуальная дока по torch / gymnasium / PySide6. Версии в проекте не зафиксированы — сверяйся с context7 вместо памяти при работе с этими API.
- **`qt-development-skills`** — любые правки/ревью Qt/QML в `app/gui_qt/`.
- **`code-simplifier`** — вычистка дублирования между параллельными реализациями (DQN/PPO/AZ/MuZero).
- **`commit-commands`** — стандартизированный commit/push/PR workflow.
- **`claude-code-setup`** — анализ репо и подбор hooks/skills/MCP под проект.
- **`remember`** — непрерывная память между сессиями (долгий проект, распределённое обучение).

## Автоматизации проекта (`.claude/`)

В репозитории настроены локальные автоматизации — учитывай их:

- **Хуки** (`.claude/settings.json` + `.claude/hooks/`):
  - `guard_paths.py` (PreToolUse) — **блокирует** Edit/Write по `runtime/logs/LOGS_FOR_AGENTS_*.md` и `runtime/state/*remote_is*`/pc2-конфигам. Формат логов правь только через код логирования, секреты — вручную/через GUI.
  - `ruff_fix.py` (PostToolUse) — после правки `.py` прогоняет `ruff check --fix` по файлу. Конфиг — `ruff.toml`. (Нужен `ruff` из `requirements_windows.txt`.)
- **Субагенты** (`.claude/agents/`) — вызывай перед коммитом:
  - `engine-regression-reviewer` — ревью диффа движка (фазы/бой/видимость/MCTS) на регрессии и инварианты.
  - `code-reviewer` — общий ревью + поиск рассинхрона между параллельными ветками (DQN/PPO/AZ/MuZero, `az_*`↔`gmz_*`).
- **Скиллы проекта** (`.claude/skills/`):
  - `logs-triage` — при любом баге сначала читает нужный `LOGS_FOR_AGENTS_*.md` (правило «отладка с логов»).
  - `run-40kai` (`/run-40kai`) — запуск train/eval/play приоритетно через Qt GUI.

## Карта кода (быстрая навигация)

- `core/engine/` — движок: фазы хода, бой, миссии, видимость, метрики, профайлер I/O.
- `core/envs/` — Gym-окружения (`warhamEnv`).
- `core/models/` — алгоритмы: `DQN.py`, `PPO.py`, `alphazero_*`, `gumbel_muzero_*`, inference/rollout протоколы и серверы.
- `core/telemetry/` — GPU/CPU телеметрия (`gpu_backend`, nvidia-ml-py, psutil).
- `app/gui_qt/` — Qt GUI (QML): старт train/eval, логи, настройки. `app/viewer/` — просмотр партий.
- `train.py` / `eval.py` / `play.py` — точки входа. Гиперпараметры — `hyperparams.json`. Reward — `reward_config.py`.
- `artifacts/` — модели (`models/`), метрики, результаты (`results/results.txt`).
- `docs/` — гайды (remote/distributed серверы). `plans/` — планы. `tests/` — тесты.

## Зависимости
